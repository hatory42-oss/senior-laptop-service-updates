# Senior Laptop Service

Компактная инженерная панель для разовой настройки Windows-ноутбука.

Целевой профиль использования: браузер, Zoom, Одноклассники и другие мессенджеры. Оптимизация отключает Xbox, карты, факс, телефонную интеграцию, телеметрию и рекламные предложения. Защитник Windows, обновления, фоновые уведомления мессенджеров, звук, камера, микрофон, Bluetooth, печать и сеть сохраняются.

Запуск: `Run-As-Administrator.cmd`.

### Windows 7

Поддерживается только Windows 7 SP1 с .NET Framework 4.5.2 или новее и Windows PowerShell 5.1 (Windows Management Framework 5.1, KB3191566). Новый запускатель проверяет эти условия до открытия панели. Если компонент отсутствует, он показывает точную инструкцию и сохраняет `SeniorLaptopService-STARTUP-ERROR.txt` рядом с программой. Файлы запускателя автоматически восстанавливаются из репозитория обновлений с проверкой SHA-256.

Безопасная очистка удаляет только временные файлы и кэши браузеров. Она не нацелена на фотографии, документы, загрузки, мессенджеры, историю, cookies, сохранённые пароли или закладки.

Исправление Яндекс.Браузера закрывает браузер, создаёт резервную копию папки Sessions и удаляет только данные восстановления открытых вкладок.

Еженедельная задача копирует минимальный сценарий в `C:\ProgramData\SeniorLaptopService` и запускает очистку по воскресеньям в 12:00. Если Zapret установлен, обновляются официальные списки доменов/IP.

Zapret скачивается только из официального репозитория `Flowseal/zapret-discord-youtube`. После установки открывается штатное меню `service.bat` для выбора и установки рабочей стратегии.

Кнопка восстановления только открывает штатную страницу Windows Recovery. Необратимый сброс не запускается без дополнительных подтверждений Windows.

## Отчёт об ошибках через GitHub

Кнопка создаёт на рабочем столе JSON с характеристиками, установленными программами, журналом панели и последними ошибками Windows, затем загружает его через GitHub API в приватный репозиторий `hatory42-oss/senior-laptop-service-reports`. Wi-Fi-пароли, история браузера, cookies, личные файлы и серийные номера не включаются.

Для загрузки используется fine-grained GitHub token, ограниченный единственным репозиторием и разрешением `Contents: Read and write`. При первой ручной отправке панель запрашивает токен и сохраняет его в скрытой папке `.secrets` рядом со сценарием — то есть на инженерной флешке, а не на обслуживаемом компьютере. После однократной настройки ручные и аварийные отчёты отправляются без повторного ввода. Нельзя загружать папку `.secrets` в публичный репозиторий или копировать её с флешки.

При любой ошибке кнопки версия 0.13.0 автоматически создаёт текстовый отчёт на рабочем столе и, если переносной токен уже настроен, немедленно загружает его в каталог `errors/ГОД/МЕСЯЦ` приватного репозитория. Если GitHub временно недоступен, локальный файл сохраняется и показывается инженеру.

## Самообновление

`Updater.json` содержит HTTPS-адрес манифеста. Манифест сообщает версию, URL сценария и SHA-256. Панель устанавливает обновление только после проверки хэша и синтаксиса, сохраняя файл `.before-ДАТА`. Укажите собственный GitHub-репозиторий вместо `CHANGE-ME`; пример манифеста находится в `update-manifest.example.json`.
